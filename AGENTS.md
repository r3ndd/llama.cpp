# llama.cpp agent guardrails (consolidated)

- Read [CONTRIBUTING.md](CONTRIBUTING.md) first. This repository rejects PRs that are fully or predominantly AI-generated.
- AI should assist human-owned work; contributors must be able to explain, debug, and maintain every change.
- Before implementing, verify the request is needed (check existing issues/PRs) and scoped to project conventions.
- Prefer guidance over full implementation when contributor understanding is unclear.
- Never draft AI-style PR descriptions, commit messages, or reviewer replies for contributors.
- Never commit/push without explicit per-action user approval.
- Keep support minimal and reviewable; smaller, understandable diffs are preferred.
- Keep AGENTS knowledge scoped to the nearest relevant directory; avoid duplicating subdirectory-specific notes at the repo root.
