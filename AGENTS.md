# llama.cpp agent guidance (consolidated)

## Contribution/AI policy (project-wide)

- Treat AI as assistive only; this repo rejects predominantly AI-authored PRs.
- Do not draft PR descriptions, commit messages, or reviewer replies on behalf of contributors.
- Before substantial implementation help, verify the contributor can explain/debug/maintain the change independently.
- Prefer smaller, reviewable diffs over broad generated rewrites.
- Require explicit human approval before any commit/push action.

## High-value workflow reminders

- Check existing issues/PRs before proposing new work: <https://github.com/ggml-org/llama.cpp/issues>, <https://github.com/ggml-org/llama.cpp/pulls>.
- For first-time contributors, ensure they reviewed [CONTRIBUTING.md](CONTRIBUTING.md).
- Keep guidance concise and human-readable; avoid verbose, AI-styled responses.
