# Optimizing C# LINQ Query Pipelines for Memory Efficiency

When working with large datasets in C#, LINQ queries that materialize intermediate collections can lead to excessive memory usage and performance degradation. This guide explains how to identify such issues and offers strategies to optimize your LINQ pipelines for better memory efficiency.

---

## 🔍 Problem: Intermediate Collection Materialization

Consider this inefficient LINQ query:

```csharp
var result = data
    .Where(x => x.IsActive)
    .ToList()           // ❌ Materializes entire filtered list
    .Select(x => x.Value)
    .ToList()           // ❌ Creates another intermediate list
    .Where(x => x > 100)
    .ToList()           // ❌ Materializes again
    .OrderByDescending(x => x)
    .ToList();          // ❌ Final materialization
```

Each `.ToList()` call creates a full in-memory copy, compounding memory usage and reducing performance.

---

## ✅ Solution: Deferred Execution with Chained Operators

Use a single `ToList()` call at the end to avoid unnecessary intermediate materializations:

```csharp
var result = data
    .Where(x => x.IsActive)
    .Select(x => x.Value)
    .Where(x => x > 100)
    .OrderByDescending(x => x)
    .ToList();
```

This ensures deferred execution, avoiding unnecessary allocations.

## 🚀 Advanced Optimization Techniques

### 1. Use `IEnumerable<T>` Instead of `ICollection<T>`

```csharp
// ❌ Memory-heavy
public IEnumerable<int> GetLargeDataset()
{
    var list = new List<int>(1_000_000);
    for (int i = 0; i < list.Count; i++) list.Add(i);
    return list;
}

// ✅ Stream-friendly
public IEnumerable<int> GetLargeDataset()
{
    for (int i = 0; i < 1_000_000; i++)
        yield return i;
}
```

Using `yield return` avoids creating large in-memory collections.

---

### 2. Batch Processing for Large Datasets

```csharp
public async Task ProcessLargeDataAsync(IEnumerable<Data> data, int batchSize)
{
    foreach (var batch in data.Buffer(batchSize))
    {
        await ProcessBatchAsync(batch);
        // Optionally clear memory after processing
    }
}

// Extension method for batching
public static class EnumerableExtensions
{
    public static IEnumerable<IEnumerable<T>> Buffer<T>(
        this IEnumerable<T> source, int size)
    {
        var buffer = new List<T>(size);
        foreach (var item in source)
        {
            buffer.Add(item);
            if (buffer.Count == size)
            {
                yield return buffer;
                buffer = new List<T>(size);
            }
        }
        if (buffer.Any())
            yield return buffer;
    }
}
```

---

### 3. Use `IAsyncEnumerable<T>` for Async Streams

```csharp
public async IAsyncEnumerable<Data> GetDataAsync()
{
    foreach (var item in largeDataset)
    {
        yield return item;
        await Task.Yield(); // Allow async processing
    }
}

// Consume asynchronously
await foreach (var data in GetDataAsync())
{
    if (data.IsActive && data.Value > 100)
        yield return data;
}
```

---

### 4. Streaming with `IQueryable<T>` for Database Queries

```csharp
// ❌ Loads all data into memory
var result = context.Users
    .Where(u => u.IsActive)
    .Select(u => u.Email)
    .ToList();

// ✅ Translates to SQL, streams results
var result = context.Users
    .Where(u => u.IsActive)
    .Select(u => u.Email)
    .AsEnumerable();
```

---

### 5. Use `Aggregate` Instead of `ToList()` + `Max`

```csharp
// ❌ Materializes entire list
var max = data
    .Where(x => x.IsActive)
    .Select(x => x.Value)
    .ToList()
    .Max();

// ✅ Single pass, no intermediate list
var max = data
    .Where(x => x.IsActive)
    .Select(x => x.Value)
    .Aggregate(0.0, (max, val) => Math.Max(max, val));
```

---

### 6. Custom Memory-Efficient Pipeline

```csharp
public class MemoryEfficientPipeline<T>
{
    private readonly IEnumerable<T> _source;

    public MemoryEfficientPipeline(IEnumerable<T> source)
    {
        _source = source;
    }

    public MemoryEfficientPipeline<T> Where(Func<T, bool> predicate)
        => new(_source.Where(predicate));

    public MemoryEfficientPipeline<T> Select<TResult>(Func<T, TResult> selector)
        => new(_source.Select(selector));

    public TResult Aggregate<TResult>(TResult seed, Func<TResult, T, TResult> func)
        => _source.Aggregate(seed, func);

    public int Count() => _source.Count();

    public T FirstOrDefault(Func<T, bool> predicate)
        => _source.FirstOrDefault(predicate);

    public async IAsyncEnumerable<T> StreamAsync()
    {
        foreach (var item in _source)
            yield return item;
    }
}
```

## 📊 Performance Comparison

| Approach                  | Memory Usage | Performance | Best For                  |
|---------------------------|--------------|-------------|---------------------------|
| Multiple `.ToList()`      | High         | Slow        | Small data sets           |
| Deferred LINQ             | Low          | Fast        | General use               |
| `yield return`            | Very Low     | Fast        | Large enumerable sources  |
| `IAsyncEnumerable`        | Very Low     | Fast        | Async/streaming pipelines |
| `IQueryable` + streaming  | Very Low     | Fast        | Database queries          |
| Batch processing          | Low          | Fast        | Large external data       |

---

## ✅ Best Practices Summary

- **Avoid `.ToList()`** unless you explicitly need a collection.
- **Use `yield return`** to stream large datasets without loading them all into memory.
- **Use `IAsyncEnumerable<T>`** for async and streaming scenarios.
- **Leverage `IQueryable<T>`** to push filtering/sorting to the database.
- **Use `Aggregate`** instead of `ToList()` + `Max`/`Sum` to reduce allocations.
- **Process in batches** for large external data sources.
- **Monitor memory usage** using tools like `GC.GetTotalMemory()` or memory profilers.
